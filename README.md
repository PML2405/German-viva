# 🇩🇪 Deutsch Viva Guide

An interactive study guide for the **German for Beginners** oral exam (Viva Voce), covering all 50 questions from the *Willkommen* 3rd Edition textbook. Built as a single HTML file — no frameworks, no install, no server needed.

**Live site →** `https://YOUR-USERNAME.github.io/deutsch-viva/`

---

## What's inside

- All **50 viva questions** with German text and English meaning
- **2–4 sample answers** per question, each with an English translation
- **Grammar tip** for every question explaining key rules
- **Search** — type any German or English word to filter instantly
- **Topic filters** — Identity, Studies, Language, Hobbies, Food, Family, Time & Plans, Dreams & More
- **Expand / Collapse all** button for quick browsing
- Fully **responsive** — works on phone, tablet, and desktop

---

## Files

```
deutsch-viva/

└── README.md       ← this file
```

## How to host on GitHub Pages (step by step)

### Step 1 — Create a GitHub account
Go to [github.com](https://github.com) and sign up if you don't have an account yet.

### Step 2 — Create a new repository
1. Click the **+** icon (top right) → **New repository**
2. Name it exactly: `deutsch-viva`
3. Set it to **Public** (required for free GitHub Pages)
4. Leave everything else as default
5. Click **Create repository**

### Step 3 — Upload your files
On the new repository page:
1. Click **uploading an existing file** (or the **Add file** → **Upload files** button)
2. Drag and drop both files:
   - `index.html`
   - `README.md`
3. Scroll down, write a commit message like `first upload`, and click **Commit changes**

### Step 4 — Enable GitHub Pages
1. Go to your repository's **Settings** tab (top menu)
2. Scroll down to the **Pages** section in the left sidebar
3. Under **Source**, select **Deploy from a branch**
4. Under **Branch**, choose `main` and leave the folder as `/ (root)`
5. Click **Save**

### Step 5 — Visit your live site
Wait about **1–2 minutes**, then go to:
```
https://YOUR-USERNAME.github.io/deutsch-viva/
```
Replace `YOUR-USERNAME` with your actual GitHub username.

> GitHub will also show you the URL at the top of the Pages settings once it's ready.

## Why `index.html`?

GitHub Pages (and all web servers) automatically serve a file named `index.html` when someone visits the root of your site. This means your URL stays clean — visitors go to `github.io/deutsch-viva/` rather than `github.io/deutsch-viva/german_viva.html`.

---

## Making updates later

1. Edit `index.html` on your computer
2. Go to your repository on GitHub
3. Click on `index.html` → click the **pencil (edit) icon**
4. Paste your updated code → click **Commit changes**

Or use **Add file → Upload files** to replace the file by uploading a new version.
